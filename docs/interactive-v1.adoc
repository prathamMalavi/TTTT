[#interactive-data-view-component]
= Replace DataViewControlsComponent

In OneCX v6, the InteractiveDataViewComponent replaces the following use cases:

* Replace **p-dataView** and **DataViewControlsComponent** template with **InteractiveDataViewComponent**
* Replace **p-table** and **DataViewControlsComponent** template with **InteractiveDataViewComponent**.

== Code Changes

* Remove `<ocx-data-view-controls>` and `DataViewControlsComponent` from `@onecx/portal-integration-angular`
* Add `InteractiveDataViewComponent` from `@onecx/angular-accelerator`.
* Replace the entire `<p-dataView>` or `<p-table>` with `<ocx-interactive-data-view>`
* Remove `DataViewModule` from `primeng/dataview` if not used elsewhere
* Add `<ocx-interactive-data-view>` and update input properties and output events as per tables below

== Examples

**Example: p-dataView & DataViewControlsComponent to InteractiveDataViewComponent**
.Before
[source, html]
----
<p-dataView [value]="items" [layout]="'grid'" [paginator]="true" [rows]="20">
  <ng-template pTemplate="header">
    <ocx-data-view-controls
      [supportedViews]="['grid']"
      [initialViewMode]="'grid'"
      [filterValue]="filterValue"
      [enableFiltering]="true"
      (filterChange)="onFilterChange($event, dataView)"
      [enableSorting]="false"
    ></ocx-data-view-controls>
  </ng-template>

  <ng-template pTemplate="gridItem" let-item>
    <div class="card">
      <div>{{ item.name }}</div>
    </div>
  </ng-template>
</p-dataView>
----

.After
[source, html]
----
<ocx-interactive-data-view
  [data]="items"
  [supportedViewLayouts]="['grid']"
  [layout]="'grid'"
  [columns]="columns"
  [filters]="filters"
  [filter]="filterValue"
  [clientSideFiltering]="true"
  [emptyResultsMessage]="'ACTIONS.SEARCH.NO_DATA' | translate"
  (filtered)="onFilterChange($event)"
  (sorted)="onSortChange($event)"
  (dataViewLayoutChange)="onDataViewChange($event)"
>
  <ng-template pTemplate="item" let-item>
    <div class="card">
      <div>{{ item.name }}</div>
    </div>
  </ng-template>
</ocx-interactive-data-view>
----

== Properties Mapping

.Input Properties (DataViewControlsComponent)
[%collapsible]
====
[.stripes-even,cols="1,1,2"]
|===
h| DataViewControlsComponent
h| InteractiveDataViewComponent
h| Notes

| supportedViews
| supportedViewLayouts
| Indicates available layouts. Example: `[supportedViewLayouts]="['list', 'grid', 'table']"`.

| initialViewMode
| layout
| Sets initial layout. Example: `[layout]="'list'"`.

| filterValue
| filter
| Filters to apply. Use the `value` key in each Filter object. Example: const filters: Filter[] = [
  { columnId: 'category', filterType: FilterType.EQUALS, value: ['books', 'electronics'] }
]

| enableFiltering
| clientSideFiltering
| Enable client-side filtering. Example: `[clientSideFiltering]="true"`

| enableSorting
| clientSideSorting
| Enable client-side sorting. Example: `[clientSideSorting]="true"`

| sortingOptions
| columns
| Configure sorting per column by setting `sortable` on each `DataTableColumn`. Example:
  const columns: DataTableColumn[] = [
    { id: 'name', nameKey: 'OCX_DATA_TABLE.COLUMN.NAME', columnType: ColumnType.STRING, sortable: true }
  ];

| defaultSortOption
| sortField
| Use the column id for the default sort field. Example: `[sortField]="'name'"`

| defaultSortDirection
| sortDirection
| Use the `DataSortDirection` enum to specify the sort direction. Example: `[sortDirection]=DataSortDirection.ASCENDING`

| columnDefinitions
| columns
| Use `id` (instead of `field`) and `nameKey` for translated column names. Control visible columns with `displayedColumnKeys`. Example:
  const columns: DataTableColumn[] = [
    { id: 'name', nameKey: 'OCX_DATA_TABLE.COLUMN.NAME', columnType: ColumnType.STRING }
  ];
  displayedColumnKeys = ['name'];

| columnTemplates
| pTemplate with selectors
| Use `p-template` with a selector. For more information, see the Template Transformation section.

| dropdownPlaceholderText
| -
| Deprecated. Use translations (i18n); there is no direct input in the new component.

| filterColumns
| filters
| Use `filters` with `clientSideFiltering`. In each filter object, provide `columnId` and `filterType`.

| translations
| -
| Use i18n / translation keys.

|===
====

.Input Properties (p-dataView)
[%collapsible]
====
[.stripes-even,cols="1,1,2"]
|===
h| p-dataView
h| InteractiveDataViewComponent
h| Notes

| value
| data
| Defines the component's data source. Example: `[data]="items"`

| paginator
| listGridPaginator, tablePaginator
| listGridPaginator and tablePaginator are enabled by default

| rows
| pageSize
| Number of items per page. Example: `[pageSize]="20"`.

| rowsPerPageOptions
| pageSizes
| Array of page size options. Example: `[pageSizes]="[20, 60, 100]"`

| alwaysShowPaginator
| -
| The paginator is shown listGridPaginator

| showCurrentPageReport
| -
| The paginator shows the current page report by default when enabled.

| currentPageReportTemplate
| -
| Use the `currentPageShowingKey` translation key to render content. The `currentPageReportTemplate` is shown when pagination is enabled. Default: `'OCX_DATA_TABLE.SHOWING'`.

| filterBy
| -
| Use `clientSideFiltering` and specify `filterType` in the Filter object.

| emptyMessage
| emptyResultsMessage
| Message displayed when no data is available.
|===

[%collapsible]

== Additional Notes

=== Key Implementation Patterns

==== Data Transformation Helper
[source, typescript]
----
public toInteractiveData(data: YourModel[]): RowListGridData[] {
  return (data ?? []).map((item, index) => ({
    ...item,
    id: item.id ?? `item-${index}`,
    imagePath: item.imageUrl ?? ''  // Required for grid/list layouts
  })) as RowListGridData[]
}
----

==== Column Definition Mapping
[source, typescript]
----
// Old Column structure
type Column = {
  field: string           // → id
  header: string          // → nameKey (translated)
  translationPrefix: string  // → combined with header into nameKey
}

// New DataTableColumn structure
const columns: DataTableColumn[] = [
  {
    id: 'status',                           // was: field
    nameKey: 'ANNOUNCEMENT.STATUS',         // was: translationPrefix + header
    columnType: ColumnType.STRING,          // NEW: required
    sortable: true,                         // NEW: enables sorting
    filterable: true                        // NEW: enables filtering
  },
  {
    id: 'priority',
    nameKey: 'ANNOUNCEMENT.PRIORITY',
    columnType: ColumnType.TRANSLATION_KEY, // For enum values
    sortable: true
  },
  {
    id: 'startDate',
    nameKey: 'ANNOUNCEMENT.START_DATE',
    columnType: ColumnType.DATE,
    dateFormat: 'dd.MM.yyyy HH:mm',         // Optional: custom date format
    sortable: true
  }
]
----

==== Generic Cell Templates (Type-Based)
[source, html]
----
<!-- For STRING columns -->
<ng-template pTemplate="stringTableCell" let-rowObject="rowObject" let-column="column">
  @switch (column.id) {
    @case ('status') { /* custom rendering */ }
    @case ('type') { /* custom rendering */ }
    @default { {{ rowObject[column.id] }} }
  }
</ng-template>

<!-- For TRANSLATION_KEY columns -->
<ng-template pTemplate="translationKeyTableCell" let-rowObject="rowObject" let-column="column">
  {{ 'ENUMS.' + column.id.toUpperCase() + '.' + rowObject[column.id] | translate }}
</ng-template>

<!-- For DATE columns -->
<ng-template pTemplate="dateTableCell" let-rowObject="rowObject" let-column="column">
  {{ rowObject[column.id] | date: column.dateFormat }}
</ng-template>
----

// State management
public displayedColumnKeys: string[] = ['status', 'name', 'date']

==== Event Handler Signatures
[source, typescript]
----
// Column visibility change
public onColumnsChange(columnKeys: string[]): void {
  this.displayedColumnKeys = columnKeys
}

// Filter change (client-side filtering handled by component)
public onFilterChange(filters: Filter[]): void {
  // Optional: Custom filter logic or persistence
}

// Sort change
public onSortChange(event: { sortColumn: string; sortDirection: DataSortDirection }): void {
  this.sortField = event.sortColumn
  this.sortDirection = event.sortDirection
}

// CRUD actions from InteractiveDataViewComponent
public onViewItem(item: RowListGridData): void {
  this.onDetail(item as YourModel, 'VIEW')
}

public onEditItem(item: RowListGridData): void {
  this.onDetail(item as YourModel, 'EDIT')
}

public onDeleteItem(item: RowListGridData): void {
  this.item4Delete = item as YourModel
  this.displayDeleteDialog = true
}
----

==== Dynamic Column Creation Helper
[source, typescript]
----
private createInteractiveColumns(): DataTableColumn[] {
  return this.oldColumns.map((col) => ({
    id: col.field,
    nameKey: `${col.translationPrefix}.${col.header}`,
    columnType: this.getColumnType(col),
    sortable: !['status', 'type'].includes(col.field),
    filterable: true,
    ...(col.isDate ? { dateFormat: this.dateFormat } : {})
  }))
}

private getColumnType(col: OldColumn): ColumnType {
  if (col.isDate) return ColumnType.DATE
  if (col.isDropdown) return ColumnType.TRANSLATION_KEY
  return ColumnType.STRING
}
----

== Common Migration Patterns

=== Property Mapping Quick Reference

[.stripes-even,cols="2,2,3"]
|===
h| Old Pattern
h| New Pattern
h| Notes

| `[value]="data"`
| `[data]="toInteractiveData(data)"`
| Add `toInteractiveData()` helper for type compatibility

| `[columns]="filteredColumns"`
| `[columns]="columns"` + `[displayedColumnKeys]="displayedColumnKeys"`
| Separate column definitions from visibility control

| `(filterChange)="onFilterChange($event, table)"`
| `(filtered)="onFilterChange($event)"`
| Component handles filtering when `clientSideFiltering` enabled

| `[paginator]="true"`
| `[tablePaginator]="true"`
| Use layout-specific paginator property

| `[rows]="10"`
| `[pageSize]="10"`
| Renamed for clarity

| `[rowsPerPageOptions]="[10,20]"`
| `[pageSizes]="[10,20]"`
| Renamed for clarity

| Custom action buttons in template
| `[additionalActions]="dataActions"`
| Use `DataAction[]` array instead of template buttons

|===

=== Common Issues & Quick Fixes

[.stripes-even,cols="2,3"]
|===
h| Issue
h| Solution

| Type error: `YourModel[]` not assignable to `RowListGridData[]`
| Add `toInteractiveData()` helper or use `$any(data)` cast

| Templates not rendering
| Use type-based templates (`stringTableCell`) with `@switch`, not column-specific templates

| Event handler errors
| Update signatures: `onFilterChange(filters: Filter[])`, `onSortChange(event: {sortColumn, sortDirection})`

| Missing COPY button
| Add to `additionalDataActions: DataAction[]` array

| Columns not sortable
| Add `sortable: true` to column definition

| Column visibility not working
| Bind `[displayedColumnKeys]` and handle `(displayedColumnKeysChange)` event

|===
